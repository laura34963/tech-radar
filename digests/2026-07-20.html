GHSA-pjwm-pj3p-43mv: axios

axios NO_PROXY bypass via IPv4-mapped IPv6 addresses; incomplete fix for CVE-2025-62718.

Read more

axios's shouldBypassProxy fails to normalize IPv4-mapped IPv6 addresses (e.g. ::ffff:127.0.0.1), so a host that should match a NO_PROXY entry is not recognized and the request is routed through the proxy anyway. This is a follow-up to the earlier CVE-2025-62718 fix, which did not cover this address form. An attacker who controls proxy configuration or can influence request targets can force traffic meant to bypass the proxy through it, enabling interception or SSRF-style routing.

Why it matters

axios is a direct dependency in your npm stack and is commonly used for server-side HTTP calls in Next.js/React and Node services, where NO_PROXY often protects internal/metadata endpoints; a bypass can leak internal traffic through an untrusted proxy or defeat egress controls.

Recommended action

Upgrade axios to the patched version once released (check the GHSA advisory for the fixed range) and run npm audit; interim, avoid relying on NO_PROXY with IPv4-mapped IPv6 hosts and pin proxy settings server-side.
